CSS rehaul

08 Jun 2021 - Fix this and I might feel like deploying

After getting the photo page working, I realized the site needed a fresh coat of CSS. A few critical problems need to addressed

  • Style of articles compared to their titles
    • I don’t want to think about what # Heading style I’m using in markdown
    • The cleanest solution involves changing the titles styles and leaving content to the default styles
    • These styles need to be chosen
  • Grid layout used for header and navbar is not behaving the way I want it to
    • The basic problem here (as I understand) is that I’ve told the nav bar and header to take up 10% of the page, and when I have a page filled with content its stretching
    • I want the header to adjust to the size of the screen, not the amount of content I have
  • SCSS has no structure
    • Currently as I move between parts of the website I’ve been custom making new CSS
    • For both the portability/size of the style-sheet, as well as consistency across the website I need to clean this up
  • Article/Album lists are not consistent
    • The different lists of types of posts (on about, photos and posts) are all different, and none are good
    • Simplify the design here and make it consistent
  • Photos should be resized for use in a post
    • When this is solved I’ll be able to demonstrate the different issues here, basically I think this can be solved by changing the image property to fit the page

Fixing the layout

I started here because presumably this is a quick fix. The pertinent problem involved changing

grid-template-rows: 5% 5% auto;

to

grid-template-rows: 5vh 5vh auto;

to scale with the window size instead of the content size.

I then realized my nav bar was moving depending on the size of the current page.
